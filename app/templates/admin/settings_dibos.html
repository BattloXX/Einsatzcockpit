Hinweis

DIBOS EventHub ist eine eigenständige, einfachere Schnittstelle als LIS/IPR (siehe LIS-Anbindung) — die Antworten sind JSON statt SOAP-XML. Diese Seite liest primär mit: es wird NIE ein Einsatz angelegt. Optional (Schalter unten, Default aus) kann ein bereits bestehender, aktiver Einsatz um Zusatzinfos ergänzt werden (Einsatzort, Einsatzcode/Diagnose, BMA-Nr., Meldungsprotokoll) — siehe Einsatz-Infos. Es gibt zwei getrennte Zugangsdaten: ein vom Betreiber vergebenes Gateway-Konto (HTTP-Basic) und das eigene Org-Servicekonto (im SOAP-Header übertragen).

Einsatz-Infos

Übersicht je Einsatz, was DIBOS zusätzlich beigetragen hat (Einsatzcode/Diagnose, BMA-Nr., Kommentar, Anzahl übernommener Meldungen) — unabhängig davon, ob gerade eine Aufzeichnung läuft.

Verbindung

Schaltet den leichten Hintergrund-Erkennungs-Loop für diese Organisation ein (fragt nur GetCurrentEvents ab)

Sobald der Erkennungs-Loop einen eigenen aktiven Einsatz sieht, startet automatisch eine vollständige Rohdaten-Aufzeichnung (siehe Diagnose-Sektion unten) — spart den manuellen Klick genau während des Einsatzfensters.

Ergänzt einen bereits bestehenden, aktiven Einsatz (per Einsatznummer zugeordnet) während der Voll-Aufzeichnung um Einsatzort, Einsatzcode/Diagnose, BMA-Nr. und das sichtbare Meldungsprotokoll aus DIBOS. Nur wirksam, solange "Voll-Tracing automatisch starten" oben aktiviert ist.
